Archetypes

Archetyped ECS

This is a personal project of mine that follow an Entity-Component-System (ECS) architecture that makes writing high performance code easy. Its 'one line' modular, so a new system or new component can be registered and start running with a single line of code. Many features get generated through reflection, such as UI and serialization.

The video is showing 125k entities running at 60fps with a 0.1% ECS overhead.

Want to talk about: - Engine Highlevel Overview - The engine aims to be a sandbox where I can efficiently create and maintain features - Currently Implemented - Spatial Audio - Serialization - Models/Animations - Collision Detection - Cubic Spines - This engine api follows an - ECS Archetypes, iteration interface - PBR algorithym, - Shadow algorithym - Keyed Vector (std::hive) - gpu_vector - Modular EngineManger - Reflection based component management.

Lighting

Physically Based Rendering

This a renderer that follows many efficient rendering techniques. It uses deffered rendering, batched rendering, and bindless rendering. It supports shadow mapping following momentum shadow smoothing, Physically-Based-Rendering(PBR), and Image-Based-Lighting(IBL). The engine also has support for both point light and directional lights.
